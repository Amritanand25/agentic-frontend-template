# Design Principles in UI

Design principles are what UX/UI designers use intentionally or subconsciously in their day-to-day workflow. These are the fundamental rules that help achieve great results in the visual, user interface, graphic, or any other kind of design. These rules are based on the Gestalt principles created by psychologists in the early 20th century in Austria and Germany.

> Fundamental design principles are the rules that designers should follow to achieve a good result and composition in their user interface designs.

---

## Core Principles

### 1. Emphasis / Dominance

Create a focal point on the screen that will intentionally draw user's attention.

**How to apply:** Use size, color, contrast, or whitespace to make the most important element stand out. In a dashboard, the primary KPI or action should immediately catch the eye before secondary information.

---

### 2. Unity / Rhythm

Create a pattern from the elements to achieve harmony and visual rhythm.

**How to apply:** Repeat consistent spacing, colors, and component styles across the interface. A well-designed data table, card grid, or navigation menu exhibits rhythm through uniform sizing and predictable intervals.

---

### 3. Hierarchy

Arrange the elements in order to define their importance and role by size, font, color, and other properties.

**How to apply:** Use typography scale (headings > subheadings > body), color weight (primary > subdued), and spatial prominence (top-left reads first) to guide users through content in the intended order.

---

### 4. Balance

Arrange the elements on the screen to achieve even distribution of shape and space.

**How to apply:** Distribute visual weight evenly -- a heavy chart on the left can be balanced by a data panel on the right. Symmetrical layouts feel stable; asymmetrical layouts feel dynamic but still need equilibrium.

---

### 5. Proportion / Scale

Create a baseline for the viewer to compare the size of elements.

**How to apply:** Use a consistent type scale and spacing system (design tokens). When one element is intentionally larger, it signals importance. Avoid arbitrary sizing -- every size difference should communicate meaning.

---

### 6. Contrast

Emphasize the difference between elements using color, shape, or size.

**How to apply:** Ensure sufficient contrast between text and background (WCAG AA minimum). Use contrasting colors for primary actions vs. secondary actions. Dark-on-light and light-on-dark pairings guide attention.

---

### 7. Similarity

Use color, shape, size, or other property to make the user perceive multiple elements as belonging to the same group.

**How to apply:** All filter pills should look alike. All action buttons in a toolbar should share the same style. When elements share visual traits, users understand they serve related functions without reading labels.

---

## Practical Tips

Make sure you evaluate your designs against each of these principles and achieve a good combination of them in your composition.

- **Emphasis:** Does the most important element stand out?
- **Unity:** Do elements feel cohesive and rhythmically consistent?
- **Hierarchy:** Can users scan the page in the intended order?
- **Balance:** Is visual weight distributed intentionally?
- **Proportion:** Are size differences meaningful, not arbitrary?
- **Contrast:** Are interactive and important elements distinguishable?
- **Similarity:** Do related elements look related?
