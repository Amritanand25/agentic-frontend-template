# 10 Usability Heuristics for UX Design

> **Usability heuristics are the principles of interaction design created by Jacob Nielsen. These principles are used in usability evaluation to identify any design issues related to user interface.**

---

## The 10 Heuristics

### 1. Visibility of System Status

System informs users about what's going on.

The system should always keep users informed about what is happening through appropriate feedback within reasonable time. Loading indicators, progress bars, success/error messages, and real-time status updates all serve this principle.

---

### 2. Match Between System and the Real World

The system should speak the users' language.

Use words, phrases, and concepts familiar to the user rather than internal jargon. Follow real-world conventions, making information appear in a natural and logical order.

---

### 3. User Control and Freedom

Support undo and redo.

Users often make mistakes or change their minds. Provide clearly marked "emergency exits" to leave unwanted states without lengthy processes. Support undo, redo, and easy navigation back.

---

### 4. Consistency and Standards

Follow platform conventions.

Users should not have to wonder whether different words, situations, or actions mean the same thing. Follow established platform and industry conventions so users can transfer their existing knowledge.

---

### 5. Error Prevention

Prevent a problem from occurring in the first place.

Even better than good error messages is a careful design that prevents problems from occurring. Eliminate error-prone conditions or check for them and present users with a confirmation option before they commit to an action.

---

### 6. Recognition Rather Than Recall

Minimize the user's memory load by making objects, actions, and options visible.

The user should not have to remember information from one part of the interface to another. Instructions for use of the system should be visible or easily retrievable whenever appropriate.

---

### 7. Flexibility and Efficiency of Use

Allow users to tailor frequent actions by making complex tasks fluid and efficient.

Accelerators -- unseen by novice users -- may speed up interaction for expert users. Allow users to customize frequent actions through keyboard shortcuts, saved preferences, and workflow optimizations.

---

### 8. Aesthetic and Minimalist Design

Get rid of redundant stuff, provide only necessary info in an elegant way.

Every extra unit of information in an interface competes with the relevant units of information and diminishes their relative visibility. Interfaces should not contain information that is irrelevant or rarely needed.

---

### 9. Help Users Recognize, Diagnose, and Recover from Errors

Each error message should:

1. Be expressed in plain language (no codes)
2. Precisely indicate the problem
3. Constructively suggest a solution

---

### 10. Help and Documentation

Provide hints to guide the user through the interface.

Even though it's better if the system can be used without documentation, it may be necessary to provide help and documentation. Such information should be easy to search, focused on the user's task, list concrete steps to carry out, and not be too large.

---

## Practical Tips

- Use usability heuristics as rules of thumb, not as strict usability guidelines.
- Run each design concept through these principles to reveal caveats and issues.
