# Design Thinking Process

> **Design Thinking is a design methodology that is used for solving design problems using a human-centric approach. It allows to tackle ill-defined and complex user problems and provide creative and effective solutions through empathizing with the user.**

The design thinking process is iterative and consists of 5 phases: Discovery, Definition, Ideation, Prototyping and Testing. Feedback is collected between each phase, and the process can loop back (iterate) at any point.

```
         ◄──────────────── I T E R A T E ─────────────────►

  collect      collect       collect       collect
  feedback     feedback      feedback      feedback
    ↑  ↓        ↑  ↓          ↑  ↓          ↑  ↓

DISCOVER ──► DEFINE ──► IDEATE ──► PROTOTYPE ──► TEST
```

---

## The 5 Phases

### 1. Discover -- Learn About the User

Understand who your users are, what their problems are, and the context they operate in.

**Activities:**

- User interviews
- Stakeholder interviews
- Surveys
- Data analysis
- Metrics
- Competitors
- Focus groups
- Observation
- Clustering insights
- Context mapping
- Customer journey maps

---

### 2. Define -- Determine Features

Synthesize research findings into clear problem statements and user needs.

**Activities:**

- Personas
- Empathy maps
- User journeys
- Storyboards
- User stories
- Problem statement
- Narratives
- Assumptions mapping
- Task analysis
- Jobs to be done
- Comparative analysis

---

### 3. Ideate -- Brainstorm Solutions

Generate a wide range of creative solutions to the defined problems.

**Activities:**

- Brainstorm sessions
- Mind maps
- Affinity maps
- Storyboard
- Card sorting
- User journeys
- User flows
- Information architecture
- Service blueprints
- Business model canvas
- Crazy 8's
- Design principles

---

### 4. Prototype -- Simulate User Experience

Build tangible representations of ideas to explore and communicate solutions.

**Activities:**

- Paper prototypes
- Micro-interactions
- Detailed user flows
- Mockups
- Interactive prototypes
- Wireframes
- High fidelity design
- Design hand-offs
- Design documentation
- HTML/JS prototypes

---

### 5. Test -- Validate with Users

Put prototypes in front of real users to validate assumptions and uncover issues.

**Activities:**

- Usability testing
- Shadowing
- A/B testing
- SUS surveys
- Heuristic evaluation
- QA
- Analytics
- Performance testing
- Observations
- Desirability evaluations
- Metrics
- Eye tracking

---

## Practical Tips

- Design process is not linear, it has a parallel and iterative nature. A designer can dive into different phases depending on a situation.
- Use visual and object languages to communicate your ideas.
- Empathize with users to understand their psychological and emotional needs and what is meaningful to them.
- Generate ideas by alternating divergent and convergent thinking.
- Rapid prototyping is a powerful technique that allows to avoid a lot of debates and establish a shared understanding in the team. Use it all the time.
