# Law of the Hammer in UX Design

> **"If the only tool you have is a hammer, you tend to treat everything as if it were a nail."**

Also known as the **Law of the Instrument**, **Maslow's Hammer**, or the **Golden Hammer**. Created by Abraham Maslow in 1966.

This is a cognitive bias that represents the effect of over-reliance on a familiar tool. The examples of the law of the hammer can be met at any point in the design process where the designers tend to try to solve a problem using the same tool or sets of tools over and over again. In a sense, it can be treated as an opposite side of consistency.

---

## How It Manifests in Design

The law can get spotted in many aspects of UX design or development process. A common example:

**Design System --> Dictates UX --> Range of Products**

A design system contains established patterns. Those patterns are then applied uniformly across a range of products. The downside of having established design patterns is limited user experience. In this case the design process becomes **pattern-centric instead of being user-centric**.

### Warning Signs

- Reaching for the same UI component for every problem without evaluating alternatives
- Forcing user flows into existing patterns rather than designing for the specific use case
- Choosing familiar tools/frameworks without assessing if they fit the current problem
- Applying the same research methodology to every project regardless of context
- Reusing the same layout template for pages with fundamentally different user goals

---

## Practical Tips

- Think outside the box and try seeking different perspectives any time you approach a new problem.
- Remove all possible limitations when you brainstorm a solution to a problem.
- Always question existing patterns and validate their value and efficiency.
- Try new techniques, methodologies.
