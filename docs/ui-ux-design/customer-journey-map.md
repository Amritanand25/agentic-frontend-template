# Customer Journey Map Template

> **Customer journey map is a popular UX exercise that allows to discover a lot of insights about users. It depicts a holistic story of user's experience from initial interaction through engaging with product and becoming a loyal customer.**

Throughout the journey a designer can reveal more details such as user's emotional transformation, pain points and UX opportunities.

---

## Journey Map Structure

The map follows the user across 5 stages, tracking 6 dimensions at each stage.

### The 5 Stages

**Awareness --> Consideration --> Acquisition --> Service --> Loyalty**

### The 6 Dimensions

1. **Steps** -- What the user is trying to do
2. **Thinking** -- What questions are on their mind
3. **Doing** -- What actions they take
4. **Feeling** -- Their emotional state
5. **Pain Points** -- Frustrations and obstacles
6. **Opportunities** -- How you can improve the experience

---

## Example: Choosing a Cable/Digital Television Provider

### Awareness

| Dimension         | Details                                                                                    |
| ----------------- | ------------------------------------------------------------------------------------------ |
| **Steps**         | Wants to find a TV provider                                                                |
| **Thinking**      | Who's on the market?                                                                       |
| **Doing**         | Wants to start researching                                                                 |
| **Feeling**       | Curious                                                                                    |
| **Pain Points**   | Is not aware of all products; doesn't know what to choose; doubts the value of the product |
| **Opportunities** | Empower word of mouth; extend marketing channels                                           |

### Consideration

| Dimension         | Details                                                                                                           |
| ----------------- | ----------------------------------------------------------------------------------------------------------------- |
| **Steps**         | Needs to choose a provider                                                                                        |
| **Thinking**      | Who is the best provider? What are the prices? What people prefer? What are the offerings? What are the features? |
| **Doing**         | Goes to the website; searches the web for feedback; compares prices; compares features                            |
| **Feeling**       | Confused / Overwhelmed                                                                                            |
| **Pain Points**   | Doesn't know where to start; doesn't want to spend a lot of time on research                                      |
| **Opportunities** | Create positive image of the product; contextual promo; decrease frictions for trial opportunities                |

### Acquisition

| Dimension         | Details                                                                              |
| ----------------- | ------------------------------------------------------------------------------------ |
| **Steps**         | Subscribes to a plan                                                                 |
| **Thinking**      | Is there a discount? Is there a trial? How to pay? How to cancel the membership?     |
| **Doing**         | Makes decision; pays for subscription; starts trial; installs the equipment/software |
| **Feeling**       | Uncertain                                                                            |
| **Pain Points**   | Can't pay by BitCoin; the payment process is unclear                                 |
| **Opportunities** | Improve payment UX; support more payment systems                                     |

### Service

| Dimension         | Details                                                                                  |
| ----------------- | ---------------------------------------------------------------------------------------- |
| **Steps**         | Uses the service                                                                         |
| **Thinking**      | How good is the quality? How good is support? How to connect a new device?               |
| **Doing**         | Uses the service; builds playlists, favourites, bookmarks; connects new devices          |
| **Feeling**       | Satisfied                                                                                |
| **Pain Points**   | Hard to build playlists; hard to find content; not enough content; buffering issues      |
| **Opportunities** | Improve playlists UX; provide better streaming quality; increase content discoverability |

### Loyalty

| Dimension         | Details                                                                                             |
| ----------------- | --------------------------------------------------------------------------------------------------- |
| **Steps**         | Extends service subscription                                                                        |
| **Thinking**      | Is there going to be more content? Is there going to be new features? Will the price change?        |
| **Doing**         | Extends subscription; recommends the service; joins the community (forum, slack channels, etc.)     |
| **Feeling**       | Happy / Loyal                                                                                       |
| **Pain Points**   | No discounts; not enough other incentives                                                           |
| **Opportunities** | Create loyalty programs; turn users into advocates; offer partner programs; improve discounts model |

## Practical Tips

- You can use Emojis to reflect the emotional states.
- Justify your assumptions based upon real data: analytical and anecdotal.
- Put yourself into customer's shoes as you think about pain points and interactions.
- Invite stakeholders and do a whiteboard workshop with them -- get the entire team involved in the process.
- Focus on only one type of persona in a map.
- If it's feasible, validate the map with a customer.
